# 不気味タロット占いサービス 要件定義書 & Claude Code実装指示書

最終更新: 2026-07-04

---

## 1. プロジェクト概要

| 項目 | 内容 |
|---|---|
| サービス名（仮） | 未定（Claude Codeへの依頼時に候補出しを依頼してもよい） |
| 目的 | 実際に一般公開して使ってもらう占いWebサービス |
| コンセプト | 不気味なタロットカードのビジュアル × 不気味なピアノBGMが流れる、没入感のある一枚引きタロット占い |
| 参考サービス | [SPUR パワータロット占い](https://spur.hpplus.jp/fortune/special/tarot/)（テーマ選択→1枚引き）、[cocoloni タロット占い ワンオラクル](https://honkaku-uranai.jp/daily/tarot/one/)（1〜3枚スプレッド） |
| 占いの形式 | SPUR型を採用。テーマを選んでから1枚引く形式（スプレッドは将来拡張） |

---

## 2. ターゲットユーザー・利用シーン

- 日常的に「今日の運勢」を確認したい一般ユーザー
- 不気味・ゴシックな世界観のコンテンツを好む層（通常の可愛い/明るいタロットサイトとの差別化）
- スマホでの利用がメインだが、PCでも快適に使えること

---

## 3. 差別化ポイント

参考2サイトが「明るく親しみやすい」トーンなのに対し、本サービスは**不気味・ゴシック・儀式的な世界観**で統一する。カードイラストではなく **SVG/CSSによる幾何学的・記号的なカードデザイン** を採用し、著作権リスクなく独自の意匠を作る。BGMには**商用利用可能なフリー音源の不気味なピアノ曲**を使用し、没入感を演出する。

---

## 4. 機能要件（MVP）

### 4.1 トップページ
- サービス名・キャッチコピー・不気味な世界観を伝えるビジュアル（SVGアニメーション背景など）
- 「占いを始める」導線
- BGMは自動再生ではなく、ユーザー操作（クリック/タップ）をトリガーに再生開始（ブラウザの自動再生制限対策、かつ演出上も「儀式の開始」感を出せる）
- ミュート/再生ボタンを常時表示（アクセシビリティ対応必須）

### 4.2 テーマ選択画面
- MVPでは **3〜5テーマ** を用意する
  - 例：総合運、恋愛運、仕事運、金運、対人運　※最終確定はClaude Codeとの壁打ち、または執筆担当者が決定
- テーマごとにアイコン（SVG）とテーマカラーの差分演出

### 4.3 カードを引く演出
- テーマ選択後、カードシャッフル→1枚引く演出（アニメーション）
- **日替わり固定ロジック**：同じ日・同じテーマであれば、同じユーザーには同じ結果を返す（日付＋テーマ名をシードにした疑似乱数）
- 正位置・逆位置もシードに含めて決定
- 「今日はもう占った」状態のUI（再訪問時に同じ結果を即表示、または「本日の結果はもう出ています」の案内）

### 4.4 結果表示画面
- 引いたカード（SVGビジュアル）、カード名、正位置/逆位置
- カード×テーマに対応した占い文（詳細は5章）
- 「もう一度占う」導線（別テーマを選び直す形。同テーマの引き直しは日替わり仕様のためNG）

### 4.5 SNSシェア機能
- 結果画面から X（Twitter）へのシェアボタン
- シェア文言に「カード名＋一言＋サイトURL」を自動生成（Web Share API、またはXの intent URL を利用。バックエンド不要でクライアント側のみで完結させる）

### 4.6 音声（BGM）
- 不気味なピアノ曲をループ再生
- ミュート切り替えボタン必須
- 画面遷移（テーマ選択→カードを引く→結果）に応じて音量やエフェクトを変化させるのは任意（余力があれば）

## 5. カード・コンテンツ仕様

### 5.1 カード構成
- **大アルカナ 22枚 + 小アルカナ 56枚（4スート×14枚）＝ 計78枚**
- 各カードに正位置・逆位置の意味を持たせる

### 5.2 データ構造（JSON設計案）

```json
// cards.json
{
  "id": "major_00",
  "name": "愚者",
  "nameEn": "The Fool",
  "arcana": "major",       // "major" | "minor"
  "suit": null,             // "wand" | "cup" | "sword" | "pentacle" | null（大アルカナはnull）
  "number": 0,
  "keywords": { "upright": ["自由", "無邪気", "冒険"], "reversed": ["無謀", "軽率"] }
}
```

```json
// fortunes.json （カード × テーマ × 正逆位置の占い文データベース）
{
  "cardId": "major_00",
  "theme": "love",
  "upright": "本文（100〜150字程度を想定）",
  "reversed": "本文（100〜150字程度を想定）"
}
```

- カード数78 × テーマ数（3〜5）× 正逆2 = **468〜780パターン** の文章が必要になる想定
- 全パターンを人力執筆するのは非現実的なため、**Claude Codeに文章生成ロジック（またはデータ自体の生成）を依頼する**

### 5.3 文章生成方針（Claude Codeが担当する範囲）
- カードの基本キーワード（keywords）× テーマ性 を組み合わせて、テンプレート・パターン合成、またはLLM生成スクリプトによって全パターンの下書きを作成する
- 不気味なトーン（断定的すぎない、示唆的・警句的な言い回し）を文体ガイドとして与える
- 生成後、人力での文言チェック・調整を前提とした運用フローにする

## 6. デザイン仕様（不気味な世界観）

### 6.1 カラーパレット（案）
| 用途 | 色 |
|---|---|
| 背景（ベース） | ほぼ黒に近いダークパープル `#0d0713` 系 |
| メインアクセント | 深紅・血赤 `#6e0d25` 系 |
| サブアクセント | くすんだゴールド `#a8892f` 系（燭台・装飾線） |
| テキスト | オフホワイト／灰白 `#e8e3da` 系 |

### 6.2 フォント
- 見出し・カード名：ゴシック/儀式的な雰囲気の欧文ディスプレイフォント（数字・英字部分のみ）＋ 和文は明朝系（Noto Serif JP等）で厳かさを演出
- 本文：可読性重視の和文フォント

### 6.3 カードビジュアル（SVG/CSS）
- イラストではなく、**幾何学模様・魔法陣風の記号・アイコン**でカードごとの意匠を表現
- 共通の額縁（フレーム）デザイン＋カードごとに中央のシンボルを差し替える構成にすると、SVG管理・実装コストを抑えられる
- ロウソクの揺らぎ・グリッチ・ノイズ等のCSSアニメーションで不穏さを演出

### 6.4 アニメーション
- カードシャッフル→引く演出（カードが裏返る3Dフリップなど）
- 結果表示時のフェードイン＋微振動やグリッチ演出

### 6.5 サウンド
- 商用利用可能なフリー音源サイト（例：DOVA-SYNDROME、甘茶の音楽工房、魔王魂 など）から「不気味・ホラー・ピアノ」等で検索し、ライセンス条件（クレジット表記要否等）を確認の上で選定
- 選定は人手で行い、音声ファイルのみをアセットとしてClaude Codeに渡す想定（生成は依頼範囲外）

## 7. 技術仕様

### 7.1 技術スタック
- **フロントエンドのみの静的サイト**（バックエンド・DB不要）
- ビルドツール：Vite（Vanilla TypeScript推奨。将来的な保守性とGitHub Actionsとの相性を考慮）
- 状態管理・演出：Vanilla JS/TSで十分（フレームワーク不要の想定。React等が必要そうであれば別途相談）
- 日替わり結果の保持：`localStorage` にその日の結果を保存（サーバー保存は行わない）

### 7.2 ディレクトリ構成案
```
/src
  /assets
    /audio        … BGMファイル
    /svg          … カード共通フレーム・シンボルSVG
  /data
    cards.json
    fortunes.json
    themes.json
  /scripts
    seed.ts       … 日付＋テーマから疑似乱数を生成
    draw.ts       … カード抽選ロジック
    share.ts      … SNSシェア機能
  /styles
    theme.css     … カラーパレット・フォント定義
  main.ts
index.html
```

### 7.3 日替わりロジック（シード生成の考え方）
- シード文字列 = `YYYYMMDD + テーマID`
- シード文字列をハッシュ化し、疑似乱数生成器（例：mulberry32等の軽量PRNG）に渡して「カードID」と「正逆位置」を決定
- サーバーを使わずクライアントの日付情報のみで完結させる（タイムゾーンずれの許容は仕様として明記しておく）

### 7.4 レスポンシブ対応
- スマホ・PC両方でレイアウト崩れがないこと（ブレークポイント設計をClaude Codeに一任してよいが、モバイルファーストで設計）

---

## 8. 非機能要件

- ページ表示速度：静的サイトのため軽量に保つ（画像を使わずSVGで表現する方針とも合致）
- アクセシビリティ：音声の自動再生禁止＋ミュートボタン必須、代替テキスト設定
- ブラウザ対応：主要モダンブラウザ（Chrome, Safari, Edge, Firefox 最新版）
- SEO：将来的な集客を見据え、OGP設定・メタタグ・意味のあるHTML構造を最低限用意する

---

## 9. デプロイ

- **GitHub Pages** を利用（無料・手軽さ優先）
- GitHub Actionsで `main` ブランチへのpushをトリガーにビルド＆デプロイを自動化（Vite build → `gh-pages` ブランチ or Pages用アーティファクトへ発行）

## 10. 開発フェーズ（Claude Codeへの依頼順序の目安）

1. プロジェクト雛形構築（Vite + TypeScript セットアップ、ディレクトリ構成）
2. カードデータ（cards.json）78枚分の作成
3. カードSVGデザイン（共通フレーム＋シンボル差分の仕組み）の実装
4. テーマ選択〜カードを引く〜結果表示の画面遷移・基本ロジック実装
5. 日替わりシードロジックの実装
6. 占い文章データ（fortunes.json）の生成ロジック・下書き作成
7. BGM再生・ミュート機能の実装
8. SNSシェア機能の実装
9. レスポンシブ・アクセシビリティ調整
10. GitHub Pages向けデプロイ設定（GitHub Actions）

## 11. Claude Codeへの指示書（プロンプト本文）

以下はそのままClaude Codeに渡すことを想定した指示文です。

```
あなたには「不気味タロット占い」という一般公開Webサービスの実装を依頼します。

【概要】
- テーマ（3〜5種類、例：総合運・恋愛運・仕事運・金運・対人運）を選ぶと、
  日替わりで固定された1枚のタロットカード（正位置/逆位置）が引かれ、
  そのテーマに応じた占い文が表示されるWebサービスです。
- 世界観は「不気味・ゴシック・儀式的」。参考の明るいタロットサイトとは逆の、
  ダークで没入感のあるトーンにしてください。

【技術要件】
- フロントエンドのみの静的サイト（バックエンド・DB不要）
- Vite + TypeScript（Vanilla、フレームワーク不使用）
- GitHub Pagesへのデプロイを前提とし、GitHub Actionsでのビルド・デプロイ自動化も設定する
- 日替わり結果は「日付＋テーマ名」をシードにした疑似乱数で決定し、
  同じ日・同じテーマなら同じユーザーには同じ結果を返す（サーバー不要、localStorageで管理）

【カード仕様】
- タロットカードは大アルカナ22枚＋小アルカナ56枚の計78枚
- カードのビジュアルは写実的なイラストではなく、SVG/CSSによる幾何学的・記号的なデザインとする
  （共通の額縁フレーム＋カードごとのシンボルを差し替える設計を提案してください）
- 各カードは正位置・逆位置それぞれにキーワードと占い文を持つ

【コンテンツ生成】
- カード×テーマ×正逆位置の占い文（全パターン、100〜150字程度）を、
  各カードの基本キーワードとテーマ性を掛け合わせて生成してください。
- 文体は断定的すぎず、示唆的・警句的で不気味なトーンにしてください。
- 生成したデータはJSON形式（fortunes.json）で管理してください。

【機能要件】
1. トップページ：世界観を伝えるビジュアルと「占いを始める」導線
2. テーマ選択画面
3. カードを引く演出（シャッフル→1枚引く、フリップアニメーション等）
4. 結果表示画面（カードビジュアル・カード名・正逆位置・占い文）
5. SNS（X）シェアボタン（クライアント側のみで完結する形、intent URL等を利用）
6. BGM再生機能：不気味なピアノ曲をループ再生。自動再生はせずユーザー操作をトリガーに開始。
   ミュート/再生切り替えボタンを常時表示すること（音源ファイルは別途用意して渡します）

【デザイン】
- カラーパレットはダークパープル系背景＋深紅・くすんだゴールドのアクセント
- 和文は明朝系フォント、装飾的な数字・記号部分はゴシック/儀式的な欧文フォントを検討
- ロウソクの揺らぎ・グリッチ・ノイズ等のCSSアニメーションで不穏な雰囲気を演出

【非機能要件】
- スマホ・PC両方でレイアウトが崩れないレスポンシブ対応（モバイルファースト）
- 音声の自動再生はブラウザ制限・アクセシビリティの観点から避け、必ずミュート操作を用意する
- 軽量な静的サイトとして、画像アセットに頼らずSVGで表現する方針を維持する

まずはプロジェクト雛形（Vite+TypeScript構成）とディレクトリ設計、
カードデータ（cards.json）の骨格から着手し、進め方について確認しながら進めてください。
```
